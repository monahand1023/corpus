# The BGE reranker

Reranking is optional and off by default. When enabled (`--rerank` on `corpus-query` / `corpus-eval`, or `rerank=True` on the Retriever), the fused hybrid results feed into a local BGE cross-encoder (`BAAI/bge-reranker-v2-m3`) that re-scores each `(query, chunk)` pair directly instead of relying on cosine similarity or BM25 rank alone.

Only the top candidates from the fused list are sent to the cross-encoder ‚Äî a `rerank_pool_size` candidate pool, 30 by default, not the whole corpus ‚Äî because a cross-encoder forward pass costs real time per pair (~5ms on an M-series Mac).

If a per-document summary has already been generated by `corpus-summarize`, the reranker scores the query against the summary plus content, giving it document-level context beyond the bare chunk text. Crucially, there's no re-embedding at query time ‚Äî the reranker never touches vectors, it's a separate scoring pass layered on top of hybrid search's fused candidates.
